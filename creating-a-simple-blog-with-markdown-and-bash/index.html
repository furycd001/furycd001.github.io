Creating a simple blog with markdown & bash.

December 23, 2021

Using only markdown, bash & a converter we can create a super simple blog. To begin you’ll want to go ahead & install some sort of markdown converter that works inside your terminal. For this I use discount & on my arch system I can easily install it with pacman -S discount.

I’ll be using discount for this post, so make sure to change any commands as needed for whatever converter you choose to use..

So before we begin you’ll want to create a blog directory inside your home folder.

mkdir ~/blog

You can create this directory anywhere, but for me my home folder makes sense..

To get started lets create a template.html & add the lines <!-- insert here --> in the middle.

<!DOCTYPE html>
<html>
<head>
<title>Startpage..</title>
<link rel="stylesheet" href="Style.css">
<meta charset="utf-8">
<base target="_blank">
</head>
<body>
<div class="hello">

<!-- insert here -->

</div>
</body>
</html>

The insert here park is where our post will be inserted later..

With the .html created we can move onto the .css. This is were we will theme our blog & you can really add anything you wish. For this post I’ve just gone & centered everything in the middle of the page.

.hello {
  position: fixed;
  top: 50%;
  left: 50%;
  width: 100%;
  transform: translate(-50%, -50%);
  font-family: Ubuntu;
  font-size: 22px;
  font-weight: 300;
  text-align: center;
}

You can save this file with whatever name you choose, as long as you remember to change the link in the .html to match..

Once you have the directory & html/css created, we can then move on to creating our script. This first piece of code is our shebang & it’s used at the start of any script to tell the operating system which interpreter to use to parse the rest of the file.

#!/usr/bin/env bash

This next part will be our convert. This will convert all our .md files to .html and then place them in a our html/ folder which we just not long created.

for i in `ls *.md`; do
  markdown $i > ~/blog/`basename $i .md`.html;
done

Now that we have all of our files converted we can start editing our html. What we need to do is surround all our new html files with <div class='entry'> and output everything into a file called entries.html.

ls ~/blog/* | xargs -n 1 awk 'BEGIN {print "<div class=\"entry\">" } {print $0} END {print "</div>"}' > entries.html

We’re almost done now as this is the final command for our script. This command will insert everything in entries.html into your template.html file and output the result into a new file, called index.html.

sed -e '/<!-- insert here/r entries.html' template.html > index.html

With that last command added to our script we can go ahead & save it, then make it executable.

chmod +x bashblog.sh

That’s our script created, so now we can test it out by creating a few simple blog posts in markdown & then run the script. If your script fails at any point, make sure to check all the paths entered in the script are correct & that you have the correct command if using a different converter.
